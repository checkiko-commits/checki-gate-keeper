# CHECKI GATE KEEPER — Web公開手順

**対象**: 中川さんと服部会長がそれぞれ別の端末からアクセスし、データを共有できるようにする
**構成**: フロントエンド（index.html・静的サイト） + バックエンド（Google Apps Script・データ保存先）

---

## 1. なぜ2ステップ必要か

今のままの `index.html` はブラウザの `localStorage` にしかデータを保存しないため、中川さんのPCと服部会長のPCでは別々のデータが見えてしまいます。そこで、

1. **Google スプレッドシート + Apps Script** を「共有データベース兼API」として1つ用意する
2. **index.html を静的サイトとして公開**し、そのAPIにアクセスさせる

という2段構成にします。どちらもGoogleアカウント／ホスティングサービスへのログインが必要なため、その部分のみ中川さんの操作でお願いします。それ以外のコードはすべて用意済みです。

---

## 2. バックエンド（Google Apps Script）のセットアップ

対象ファイル: `Code.gs`（このフォルダ内）

1. [sheets.google.com](https://sheets.google.com) で新しいスプレッドシートを作成する（例：「CHECKI Gate Keeper データ」）
2. メニュー「拡張機能」→「Apps Script」を開く
3. デフォルトで入っているコードを全部消して、`Code.gs` の中身を貼り付ける
4. 保存する（Ctrl+S / Cmd+S）
5. スプレッドシートのタブに戻って**再読み込み**すると、メニューバーに「🔒 Gate Keeper」が出現する
6. 「🔒 Gate Keeper」→「📋 データシートを初期化する」を実行する
   - 初回は権限の許可を求められるので許可する
   - 「案件データ」という名前のシートが自動で作られる
6.5. 「🔒 Gate Keeper」→「✉️ 通知テストを送信」を実行する
   - 初回はGmail送信権限の許可を求められるので許可する（これをやっておかないと、後で案件提出・承認の通知メールが無言で失敗します）
   - `brands.masaki@gmail.com` 宛にテストメールが届けば成功
7. 右上の「デプロイ」→「新しいデプロイ」をクリック
   - 種類の選択（歯車アイコン）→「ウェブアプリ」
   - 説明：任意（例：v1）
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
   - 「デプロイ」をクリック
8. 発行された URL（`https://script.google.com/macros/s/.../exec` の形）を**コピーしておく**

> ⚠️ このURLと `Code.gs` 冒頭の `API_TOKEN` の2つがあれば誰でもデータを読み書きできてしまいます。他人に共有しないでください。

### コードを後から直した場合
「デプロイ」→「デプロイを管理」→ 鉛筆アイコン →バージョンで「新バージョン」を選んで「デプロイ」を押さないと、修正が本番URLに反映されません（Apps Scriptのよくある落とし穴です）。

## 3. フロントエンド（index.html）側の設定

`index.html` を開き、`<script>` タグの中ほどにある以下の行を書き換えます。

```js
var API_URL = "";
```

↓ 手順2でコピーした URL を貼り付けます。

```js
var API_URL = "https://script.google.com/macros/s/xxxxxxxxxxxx/exec";
```

`API_TOKEN` は `Code.gs` 側とすでに同じ値が入っているので、そのままで問題ありません（変更したい場合は両方のファイルで同じ値に揃えてください）。

設定すると、サイドバーに「⟳ 更新」ボタンと同期状況（✓ 同期済み / ⚠ 同期エラー）が表示されるようになります。`API_URL` が空のままだと、これまで通り単独モード（この端末のみ）で動作します。

## 4. 静的サイトとして公開する

`index.html` は他のファイルに依存しない1枚のHTMLなので、そのまま静的ホスティングにアップロードするだけで公開できます。おすすめは以下のどちらかです。

### 選択肢A：Netlify Drop（もっとも簡単・アカウント登録すら不要で試せる）
1. ブラウザで [app.netlify.com/drop](https://app.netlify.com/drop) を開く
2. `checki-gate-keeper` フォルダ（`index.html` が入っているフォルダ）をそのままドラッグ＆ドロップする
3. 数秒で `https://ランダムな文字列.netlify.app` のURLが発行され、即座にアクセス可能になる
4. 継続的に同じURLで使いたい場合は、画面の案内に従ってNetlifyアカウントでログイン（または新規登録）し、このサイトを「Claim」する

### 選択肢B：GitHub Pages（GitHubアカウントをお持ちの場合）
1. GitHubで新しいリポジトリを作成する（Public）
2. `checki-gate-keeper` フォルダの中身（`index.html`、`Code.gs`、`公開手順.md`）をアップロードする
   - GitHubの「Add file」→「Upload files」からドラッグ＆ドロップでOK（コマンド操作は不要）
3. リポジトリの「Settings」→「Pages」→ Source を「Deploy from a branch」、Branch を「main」に設定して保存
4. 数分後に `https://ユーザー名.github.io/リポジトリ名/` でアクセスできるようになる

いずれの場合も、`Code.gs` はアップロードしても実害はありません（`API_TOKEN` は書き換え済みの前提のため）が、気になる場合は公開フォルダから `Code.gs` と `公開手順.md` を除いて `index.html` だけをアップロードしても構いません。

## 4.5 メール通知について

以下のタイミングで自動的にメールが送られます（`Code.gs` の `notifyOnTransition_` が担当）。

| タイミング | 宛先 | 内容 |
|---|---|---|
| 中川さんが会長へ提出した時 | `brands.masaki@gmail.com`（服部会長） | 案件名・AIスコア・アプリへのリンク |
| 会長が承認した時 | `akiko.nakagawa@checki.jp`（中川さん） | 案件名・判定日時 |
| 会長が却下した時 | `akiko.nakagawa@checki.jp`（中川さん） | 案件名・修正コメント・判定日時 |

宛先を変更したい場合は `Code.gs` 冒頭の `CHAIRMAN_EMAIL` / `NAKAGAWA_EMAIL` を書き換えて、新バージョンとして再デプロイしてください。

> ⚠️ 初回のみ、Apps Scriptエディタで「🔒 Gate Keeper」→「✉️ 通知テストを送信」を実行してGmail送信権限を許可しておく必要があります（手順2の6.5参照）。これをやらないと通知メールは無言で失敗します。

## 5. 公開後の使い方

- 中川さん：ご自身の端末で公開URLを開き、案件エントリー→AI事前採点→提出、まで行う
- 服部会長：同じ公開URLを別の端末で開くと、サイドバーに提出済み案件が表示される（表示されない場合は「⟳ 更新」ボタンを押す）
- お互いの変更はタブを切り替えたタイミングと「⟳ 更新」ボタンで反映されます（自動のリアルタイム同期ではありません）

---

## 6. 困ったときは

| 症状 | 原因の可能性 | 対処 |
|---|---|---|
| 「⚠ 同期エラー」が出続ける | API_URLの設定ミス、デプロイのアクセス権限が「全員」になっていない | 手順2・3を再確認 |
| Code.gsを直したのに反映されない | 新バージョンとして再デプロイしていない | 「デプロイを管理」から新バージョンを作成 |
| 会長側に案件が表示されない | まだ同期されていない | サイドバーの「⟳」を押す |
| 提出・承認しても通知メールが来ない | 「✉️ 通知テストを送信」で権限許可をしていない、または再デプロイ漏れ | 手順2の6.5を実行→新バージョンとして再デプロイ |

## 7. 判断基準ナレッジ機能

会長との壁打ちで固まった判断基準・考え方をmdファイルとしてアップロードしておくと、各案件の「AI事前採点」タブに、その内容がそのまま参照資料として表示されるようになります。

AIには一切読ませていません。粗利率70%未満は受理不可、といった**数式による機械採点・合否ゲートはこれまで通り**で、ナレッジ機能はそこに何も手を加えません。アップロードした文章を、判断する本人（会長・中川さん）がその場で読めるようにするだけの、参照棚のような位置づけです。外部APIの契約・課金は一切発生しません。

### 7.1 Code.gsの再デプロイ

ナレッジ機能を追加したことで `Code.gs` の内容が更新されています。既存のコードをこのリポジトリの最新版で上書きし、「デプロイ」→「デプロイを管理」→ 鉛筆アイコン → バージョン「新バージョン」→「デプロイ」を実行してください（手順2参照）。

既存の案件データは自動的に引き継がれます（初期化のやり直しは不要です）。

### 7.2 判断基準ドキュメントをアップロードする

1. アプリのサイドバーにある「📚 判断基準ナレッジ」を開く
2. 「mdファイルを選択」から、会長との壁打ち内容をまとめたmdファイルを選ぶ（複数可）
3. アップロードされたファイルは中川さん・服部会長のどちらの端末からも共有される

不要になったファイルは、一覧の「削除」ボタンから削除できます。

### 7.3 案件エントリー時に参照する

1. 案件エントリーを保存し、「AI事前採点」タブを開く
2. 「判断基準ナレッジ（参照）」カードに、登録済みの全ファイルがタイトルだけ並んでいる
3. タイトルをクリックすると本文が展開され、判断材料としてその場で読める

| 症状 | 原因の可能性 | 対処 |
|---|---|---|
| 「この機能はサーバー同期（API_URL）設定後に使えます」と出る | API_URLが未設定 | 手順3を実施 |
| ナレッジのアップロード・削除ができない | Code.gsが旧バージョンのまま | 7.1の再デプロイを実施 |
